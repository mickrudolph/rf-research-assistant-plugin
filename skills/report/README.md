# RF Research Report Skill - Distribution Package

**Version:** 1.0
**Last Updated:** March 26, 2026
**Packaged For:** RainFocus Team Distribution

---

## What This Skill Does

Transforms comprehensive research synthesis documents (10-12 pages) into executive-ready stakeholder reports (2-3 pages).

**Key Features:**
- **Inverted structure** - Recommendations first, evidence last
- **SCQA framework** - Clear narrative arc (Situation → Complication → Question → Answer)
- **Confidence-weighted findings** - HIGH/MODERATE/EMERGING levels
- **Interactive HTML output** - Collapsible sections, validation matrices, RF branding
- **Markdown fallback** - Version-control friendly text format

### ✅ **Fully Self-Contained**

This package has **no external dependencies**:
- RF logo is **embedded as base64** (loaded at build time by the Python script)
- HTML generation uses `generate_html.py` (pure Python stdlib — no pip install needed)
- CSS is inlined into the final HTML automatically
- Works on any system with Python 3 and Claude Code installed

## Usage

Once installed, invoke the skill with:

```
/rf-research:report
```

Or in natural language:
```
Create a stakeholder report from the testing & versioning synthesis
```

**The skill will:**
1. Ask for your synthesis document path (if not provided)
2. Ask for output format preference (HTML or Markdown)
3. Generate the report with SCQA structure, confidence levels, and killer quotes
4. Save both HTML and markdown versions
5. Ask if you want to add any missing information (optional)

---

## Output Formats

### Format 1: Markdown Only
- Text-based report: `report-[topic]-[date].md`
- Best for version control and quick sharing
- Strong visual hierarchy even in plain text

### Format 2: RF-Branded HTML (Recommended)
- Interactive HTML: `report-[topic]-[date].html`
- Includes:
  - RainFocus branding (Berry #E71356, Grape #5C00DC)
  - Embedded RF logo
  - Collapsible evidence sections
  - Interactive validation matrix with hover tooltips
  - Confidence badges
  - Print-optimized styles
- Also saves markdown version for reference

## Design Principles

The skill follows research-backed reporting principles:

1. **Inverted Structure (Minto Pyramid)** - Lead with the answer, not the journey
2. **Three-Layer Compression** - Glance (0.5 page) → Scan (1.5 pages) → Deep Dive (link to synthesis)
3. **SCQA Framework** - Situation, Complication, Question, Answer
4. **Confidence Over Severity** - Data-driven confidence levels with participant counts
5. **"So What" Bridge** - Every finding translates to business implications
6. **Participant Voice** - Killer quotes that make findings feel real

## Troubleshooting

**Issue:** Skill not showing up in Claude Code
**Fix:** Restart Claude Code or run `/skills reload`

**Issue:** HTML output has broken logo
**Fix:** The logo is embedded in this package. If you see a broken image, the template may not have loaded correctly. Contact Mick Rudolph.

**Issue:** HTML generation takes a long time or freezes
**Fix:** HTML is now generated by `generate_html.py` (a deterministic Python script), not the LLM. It runs in under 1 second. If you still see issues, run the script manually: `python3 ~/.claude/skills/rf-research-report/generate_html.py <path-to-report.md>`
